# 🦗 电子斗蛐蛐

多角色 AI 协作对抗系统——让不同视角的 AI 角色轮流审视同一个项目，互相博弈、互相补充。

## 使用方式

在 Claude Code 中输入：

```
做一个记账App
```

系统会自动进入电子斗蛐蛐模式，依次经过：

1. **需求收集** — 标准化你的需求
2. **核心循环** — 架构师→工程师→测试→产品→总结→批判（最多3轮）
3. **用户评审** — 5个不同视角的用户角色独立评审
4. **最终整合** — 汇总意见，生成代码和报告

## 角色列表

### 核心角色（固定6个）

| 角色 | 性格 | 职责 |
|------|------|------|
| 🏗️ 架构师 A | 全局思维，爱画框图 | 技术方案、技术选型（至少2个候选）、模块划分 |
| 👨‍💻 工程师 B | 务实直接，"别废话先跑起来" | 写代码、自测通过再提交 |
| 🧪 测试员 C | 毒舌精准，"这里有个坑你看到没" | 找bug（P0/P1/P2）、跑测试、bug追踪 |
| 📋 产品经理 D | 用户导向，"你妈会用这个吗" | 业务评审、具体场景分析、差异化建议 |
| 📝 总结者 E | 冷静克制，像会议记录员 | 汇总意见、整理待办 |
| 🔥 批判者 F | 戏剧化，"等等，你们都忽略了…" | 质疑一切，但必须有理有据 |

### 用户评审角色（5个）

| 角色 | 视角 | 关注点 |
|------|------|--------|
| 🛵 外卖骑手 | 高频手机、碎片时间 | 能不能单手操作、3秒内完成核心动作 |
| 💼 白领 | 效率导向 | 有没有快捷操作、能不能批量处理 |
| 👴 退休老人 | 技术不熟练 | 字够不够大、流程够不够短 |
| 🎓 大学生 | 预算敏感、爱尝鲜 | 免费吗、有什么酷功能 |
| 👦 10岁小孩 | 完全不懂技术 | 能不能自己搞懂怎么用 |

## 收敛规则

```javascript
consecutive_clean_rounds = 0

for round in 1..3:
    bugs = C.find_new_bugs()
    objections = F.find_new_objections()

    if bugs == 0 AND objections == 0:
        consecutive_clean_rounds += 1
    else:
        consecutive_clean_rounds = 0  # 重置！

    if consecutive_clean_rounds >= 2:
        break  # 收敛
```

## 项目结构

```
cricket-battle/
├── README.md          # 本文件
├── roles.js           # 角色定义（prompt + 性格）
├── config.js          # 配置（轮数、收敛条件等）
├── report.js          # 报告生成器
└── battle.js          # 核心工作流
```

## 输出

每次运行生成：
- `output/report.md` — 完整的斗蛐蛐报告（含决策摘要、时间线、各轮详情、用户评审、未解决分歧）
- `output/` — 代码文件（由工程师角色写入）

## 配置

编辑 `config.js` 调整参数：

```javascript
MAX_CORE_ROUNDS: 3,           // 最大核心循环轮数
CONVERGENCE_THRESHOLD: 2,     // 连续N轮无问题则收敛
USER_REVIEW_COUNT: 5,         // 用户评审角色数量
```
